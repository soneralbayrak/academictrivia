3.3 Haskell as category

Let us analyze Haskell from a category theory point of view. We will consider two categories:

C = {Int,Bool,Char} (3.9) D = {Double,Bool} (3.10)

where the objects are standard Haskell types. We can then write

End(Char) = Hom(Char,Char) = {toUpper,mirror,∖x →’a’,… ⁡} (3.11) Aut(Char) = {mirror,… ⁡} (3.12) Hom(Int,Bool) = {isLeapY ear,∖x → x > 2,… ⁡} (3.13)

For instance, the function f = (∖x → x > 2) ∈ Hom(Int,Bool) is not an monomorphism, because

α = ∖x →if x is before k in alphabet 5 else 10 ∈ Hom(Char,Int) (3.14) α′ = ∖x →if x is before k in alphabet 7 else 12 ∈ Hom(Char,Int) (3.15)

even though α≠α′, we have

fα˙ = fα˙′ = ∖x → True (3.16)

however it is an epimorphism as for any βi ∈ Hom(Bool,Char) with

β(True) = b1,β(False) = b2 (3.17) β′(True) = b1′,β′(False) = b2′ (3.18) (3.19)

b1,b2,b1′,b2′ ∈ Char, we have

βḟ = ∖x →if x > 2 then b1, else b2 (3.20) β′ḟ = ∖x →if x > 2 then b1’, else b2’ (3.21)

hence β≠β′ →β.f≠β′ḟ.
